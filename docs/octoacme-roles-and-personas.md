# OctoAcme Personas

This document defines typical roles and responsibilities used in OctoAcme project docs and exercises.

---

## Developers

### Role Summary
Developers design, build, test, and deliver software components. They collaborate with product and project leads to implement features that meet acceptance criteria and quality standards.

### Responsibilities
- Implement features and fixes to meet acceptance criteria
- Write and maintain tests and documentation
- Participate in design and code reviews
- Assist in estimating and planning work
- Help identify technical risks and propose mitigations

### Goals
- Deliver reliable, maintainable code
- Reduce cycle time from idea to production
- Maintain high test coverage and observability

### Typical Communication
- Daily standups and sprint planning
- PR descriptions and code review comments
- Technical design docs when needed

---

## Product Managers

### Role Summary
Product Managers define what should be built to deliver customer and business value. They own the product vision, prioritize the backlog, and measure outcomes.

### Responsibilities
- Define problem statements and success metrics
- Prioritize the roadmap and backlog
- Collaborate with stakeholders and engineering on trade-offs
- Validate solutions through user research and metrics

### Goals
- Maximize customer value and impact
- Make clear, data-driven prioritization decisions
- Ensure product-market fit and usability

### Typical Communication
- Weekly alignment with PM and engineering leads
- Roadmap updates and stakeholder briefings
- Acceptance criteria and feature specs

---

## Project Managers

### Role Summary
Project Managers coordinate delivery activities, manage schedules, risks, and communications. They enable the team to deliver on commitments efficiently.

### Responsibilities
- Create and maintain project plans and timelines
- Manage risks, dependencies, and resource constraints
- Facilitate meetings (kickoff, planning, retrospectives)
- Ensure consistent project documentation and status reporting
- Coordinate cross-team and stakeholder communication

### Goals
- Deliver projects on time and within scope
- Minimize unplanned work and escalations
- Maintain transparency and alignment across stakeholders

### Typical Communication
- Weekly status updates and stakeholder reports
- Risk registers and decision logs
- Coordination via project boards and meeting facilitation

---

## Technical Writer

### Role Summary
Technical Writers maintain and improve documentation quality, ensuring clarity, completeness, and accessibility for all team members.

### Responsibilities
- Maintain documentation, ensure clarity and completeness
- Collaborate with developers for documentation updates
- Create and maintain technical guides, API docs, and process documentation
- Review and edit content for consistency and accuracy
- Establish documentation standards and best practices

### Goals
- Ensure all documentation is current, accurate, and accessible
- Reduce documentation debt and improve discoverability
- Enable self-service for common questions and processes

### Typical Communication
- Works alongside Developers and Project Managers to keep process docs current and clear
- Documentation review sessions
- Collaborative editing and feedback cycles

---

## UX Researcher

### Role Summary
UX Researchers gather user feedback and insights to guide product decisions and improve usability.

### Responsibilities
- Gather user feedback through interviews, surveys, and usability studies
- Recommend improvements based on usability testing and research findings
- Facilitate user studies and synthesize findings into actionable insights
- Collaborate with designers to validate design decisions
- Track user satisfaction and experience metrics

### Goals
- Understand user needs and pain points deeply
- Influence product direction with data-driven insights
- Improve overall user satisfaction and product usability

### Typical Communication
- Partners with Product Managers and Developers to guide design iterations and prioritize backlog items
- Research readouts and insight presentations
- User feedback synthesis and recommendations

---

## Security Lead

### Role Summary
Security Leads identify and mitigate security risks throughout the project lifecycle, ensuring secure development and deployment practices.

### Responsibilities
- Identify and mitigate security risks throughout the project lifecycle
- Review code and deployment steps for security compliance
- Educate the team on security best practices and threat modeling
- Conduct security assessments and vulnerability scanning
- Define and enforce security policies and standards
- Respond to security incidents and coordinate remediation

### Goals
- Minimize security vulnerabilities and risks
- Ensure compliance with security standards and regulations
- Build a security-aware culture across the team

### Typical Communication
- Works cross-functionally with Developers and Project Managers during planning and release phases
- Security review sessions and threat modeling workshops
- Incident response coordination

---

## DevOps Engineer

### Role Summary
DevOps Engineers set up and maintain CI/CD pipelines, manage deployment workflows, and ensure reliable infrastructure.

### Responsibilities
- Set up and maintain CI/CD pipelines for automated builds and deployments
- Manage deployment workflows and infrastructure as code
- Support release and rollback procedures
- Monitor system health, performance, and reliability
- Implement and maintain observability tools
- Optimize deployment processes for speed and safety

### Goals
- Enable fast, safe, and reliable deployments
- Minimize deployment-related incidents and downtime
- Improve developer productivity through automation

### Typical Communication
- Coordinates with Developers and Project Managers to ensure quality gates and deployment readiness
- Release planning and post-mortem sessions
- Infrastructure and tooling updates

## How these personas are used in the exercise
- Use these persona definitions to frame scenarios and sample interactions in the Skills Exercise.
- Each persona can be used as a persona prompt for Copilot Spaces to shape role-specific guidance.
- The expanded persona set (Developers, Product Managers, Project Managers, Technical Writer, UX Researcher, Security Lead, DevOps Engineer) provides comprehensive role definitions for improved clarity, accountability, and cross-team collaboration.
